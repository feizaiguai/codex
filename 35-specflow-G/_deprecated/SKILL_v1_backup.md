## 🎯 核心目标

**SpecFlow** 是生产级规格文档编写工具，专为**复杂企业任务**设计。

综合了主流开发方法论（TDD/BDD/DDD/ATDD/FDD/SDD）和规格工具（OpenSpec/SpecKit/BMAD）的最佳实践。

**重要**：所有输出使用**简体中文**。

---

## 🚀 使用方法（Slash Command）

### 命令格式

```
/spec [任务描述]
```

### 使用示例

```
/spec 构建 B2B 电商平台，支持多租户架构、AI 推荐、实时库存同步。预算 120 万，18 个月完成
```

Claude 会自动：
1. 分析任务复杂度  
2. 选择深度级别（快速/标准/全面）  
3. 生成 6D 规格文档  
4. **使用简体中文输出**

---

## 🔥 核心能力

### 1. 6D 规格框架（独创）

| 阶段 | 重点 | 输出 |
|------|------|------|
| **发现** Discover | 业务与利益相关者分析 | 业务背景 |
| **定义** Define | 需求与约束规格 | 需求规格 |
| **设计** Design | 领域建模与架构 | 设计规格 |
| **分解** Decompose | 工作分解与规划 | 实施计划 |
| **声明** Declare | 测试与验证策略 | 测试策略 |
| **文档** Document | 完整规格文档 | 完整规格 |

### 2. 集成方法论

- **BDD**（行为驱动）→ Given-When-Then 场景  
- **DDD**（领域驱动）→ 限界上下文、实体  
- **TDD**（测试驱动）→ 测试规格  
- **ATDD**（验收测试驱动）→ 验收标准  
- **FDD**（特性驱动）→ 特性分解  
- **SDD**（规格驱动）→ 规格优先方法

### 3. AI 驱动验证

- 冲突检测：检测跨阶段需求冲突  
- 完整性评分：衡量规格覆盖率 (0-100)  
- 一致性检查：验证术语和数据一致性  
- 可行性分析：评估技术和资源可行性

## 💡 何时使用

### ✅ 最适合

- 复杂企业项目（10+ 功能，> 3 个月）  
- 高风险项目（新产品、迁移、重构）  
- 多利益相关者项目  
- 需要全面规格才能实施的项目

### ⚠️ 不推荐

- 简单 CRUD 应用（< 5 功能）  
- 概念验证/原型  
- 已定义清楚的简单任务

**经验法则**：复杂度 > 200 小时 或 团队 > 5 人 → 使用 SpecFlow
